```javascript
// 数组是对象
typeof [1, 2, 3] === 'object'; // true

// 数组使用数字索引
const arr = [1, 2, 3];
arr[0]; // 1
arr[1]; // 2

// 数组有 length 属性
arr.length; // 3

// 数组是可迭代对象

	console.log(item);

```

### 手写数组实现

```javascript
// 手写数组构造函数
function MyArray(...args) {
	// 使用对象存储数据
	for (let i = 0; i < args.length; i++) {
		this[i] = args[i];
	}

	// length 属性自动维护
	Object.defineProperty(this, 'length', {
		get() {
			/* 计算长度 */
		},
		set(newLength) {
			/* 更新长度 */
		},
	});
}

// 实现可迭代协议
MyArray.prototype[Symbol.iterator] = function () {
	// 返回迭代器
};
```

## 🔍 关键知识点

### 1. 数组是对象

数组在 JavaScript 中本质上是对象，使用数字索引作为键名。这解释了为什么：

- `typeof [] === 'object'`
- 可以给数组添加非数字属性
- 数组可以像对象一样使用

### 2. length 属性的特殊性

数组的 `length` 属性是特殊的：

- 自动维护：当添加或删除元素时，`length` 会自动更新
- 可写：可以设置 `length` 来截断或扩展数组
- 不可枚举：`length` 不会出现在 `for...in` 循环中

### 3. 可迭代协议

数组实现了可迭代协议，因此可以：

- 使用 `for...of` 循环
- 使用扩展运算符 `[...arr]`
- 使用 `Array.from()` 等方法

### 4. 性能优化

官方数组实现有大量性能优化：

- **连续内存**：大多数情况下使用连续内存块存储
- **类型推断**：根据元素类型选择最优存储方式
- **JIT 优化**：运行时编译优化
- **原生方法**：使用 C++/Rust 等原生代码实现

## ❓ 常见问题

### Q: 数组真的是对象吗？

A: 是的，数组在 JavaScript 中本质上是对象。`typeof [] === 'object'` 返回 `true`。数组使用数字索引作为键名，并实现了特殊的 `length` 属性。

### Q: 为什么手写数组性能比官方数组差？

A: 官方数组是引擎原生实现，使用连续内存、类型推断、JIT 编译等优化。手写数组使用普通对象，需要哈希查找，无法享受这些优化。

### Q: 手写数组有什么实际用途？

A: 手写数组主要用于学习和理解数组的本质。在实际项目中，应该使用官方数组。但在某些特殊场景下，可能需要自定义数组实现。

### Q: 数组的 length 属性是如何自动维护的？

A: 数组的 `length` 属性是特殊的属性描述符。当访问数组元素或修改数组时，引擎会自动更新 `length` 值。在手写实现中，我们使用 getter/setter 来模拟这个行为。
