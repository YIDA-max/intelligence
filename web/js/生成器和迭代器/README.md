## 📖 内容结构

### 第一部分：迭代器（Iterator）

#### 什么是迭代器协议

- 迭代器是一个协议（Protocol），定义了如何遍历数据集合
- 任何实现了迭代器协议的对象都可以被迭代
- 迭代器协议要求对象必须有一个 `next()` 方法
- `next()` 方法返回：`{ value: any, done: boolean }`

#### 手动实现迭代器

- 创建一个包含 `next()` 方法的对象
- 管理内部状态（如索引）
- 返回迭代结果

#### 使用 Symbol.iterator

- 实现 `Symbol.iterator` 方法使对象可迭代
- 可以使用 `for...of` 循环遍历
- 可以使用扩展运算符 `[...]`

#### 内置可迭代对象示例

- 数组（Array）
- 字符串（String）
- Map 和 Set
- 其他内置可迭代对象

### 第二部分：生成器（Generator）

#### 生成器函数语法

- 使用 `function*` 声明生成器函数
- 生成器函数返回一个生成器对象
- 生成器对象实现了迭代器协议

#### yield 关键字的使用

- `yield` 用于暂停函数执行
- `yield` 是双向的：可以返回值，也可以接收值
- 每次调用 `next()` 从上次暂停的地方继续

#### 生成器的暂停/恢复机制

- 生成器可以暂停执行并保存状态
- 通过 `next()` 方法恢复执行
- 状态保存在堆内存中，不会丢失

#### 生成器作为迭代器

- 生成器返回的对象实现了迭代器协议
- 可以直接用于 `for...of` 循环
- 可以手动调用 `next()` 方法

### 第三部分：两者的关系和区别

#### 生成器是创建迭代器的工具

- 生成器是迭代器的一种实现方式
- 生成器自动管理状态，比手动实现更简单
- 生成器返回的对象也是迭代器

#### 对比示例

- 手动实现迭代器 vs 使用生成器
- 代码复杂度对比
- 使用场景对比

### 第四部分：实际应用场景

#### 无限序列生成

- 生成器非常适合生成无限序列
- 惰性求值：只在需要时才计算
- 示例：斐波那契数列生成器

#### 异步控制（配合 Promise）

- 生成器可以用于异步流程控制
- 这是 `async/await` 出现之前的方案
- 需要执行器函数来运行生成器

#### 数据流处理

- 管道式数据处理
- 组合多个生成器
- 实现函数式编程模式

## 💡 核心概念

### 迭代器（Iterator）

```javascript
// 迭代器协议
{
  next() {
    return { value: any, done: boolean };
  }
}
```

### 生成器（Generator）

```javascript
// 生成器函数
function* myGenerator() {
	yield 1;
	yield 2;
	yield 3;
}

// 使用
const gen = myGenerator();
gen.next(); // { value: 1, done: false }
gen.next(); // { value: 2, done: false }
gen.next(); // { value: 3, done: false }
gen.next(); // { done: true }
```

## ❓ 常见问题

### Q: 生成器和迭代器有什么区别？

A: 迭代器是协议/接口，生成器是创建迭代器的工具。生成器返回的对象实现了迭代器协议。

### Q: 什么时候使用生成器？

A: 当你需要生成序列、处理异步流程、或实现惰性求值时，生成器是很好的选择。

### Q: async/await 和生成器有什么关系？

A: `async/await` 底层基于生成器的思想，是生成器 + Promise 的语法糖。
