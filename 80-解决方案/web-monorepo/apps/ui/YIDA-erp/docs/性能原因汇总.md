### [P-001] useMultipleDicts 重复注册相同 key 导致接口重复请求

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：`useMultipleDicts` 的配置数组中，`productManageId`、`shopOption`、`currencyOption`、`operatorInfo` 四个 key 各注册了两次，导致对应接口在页面初始化时被请求两次。
- 影响：造成不必要的网络请求，增加服务端压力，拖慢页面首屏加载速度。
- 错误示例：

```ts
{ key: 'productManageId', fetchFn: getUserListByCondition, params: { lockFlag: 0 } },
// ... 其他配置 ...
{ key: 'productManageId', fetchFn: getUserListByCondition, params: { lockFlag: 0 } }, // 重复！
```

- 正确示例：

```ts
// 每个 key 只注册一次，合并注释说明用途
{ key: 'productManageId', fetchFn: getUserListByCondition, params: { lockFlag: 0 } }, // 人员列表 / 负责人
```

- 对应规范：见「二、项目编写规范 → R-001」

### [P-002] 模板内联复杂计算表达式，每次渲染重复执行

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：`el-table` 的列模板中直接写 `Array.map().join()` 等链式操作，Vue 每次重新渲染该行时都会重新执行这些表达式。
- 影响：列表行数越多，渲染开销越大，滚动/排序/筛选时会有明显卡顿。
- 错误示例：

```html
<span>
	{{ row.platforms.length <= 2 ? row.platforms.map((p: any) =>
	p.platformText).join(', ') : row.platforms.slice(0, 2).map((p: any) =>
	p.platformText).join(', ') + '...' }}
</span>
```

- 正确示例：

```ts
// 抽取为独立函数，模板只做调用
const formatPlatformText = (platforms: any[]): string => {
	if (!platforms?.length) return '';
	const texts = platforms.map((p) => p.platformText);
	return texts.length <= 2
		? texts.join(', ')
		: texts.slice(0, 2).join(', ') + '...';
};
```

```html
<span>{{ formatPlatformText(row.platforms) }}</span>
```

- 对应规范：见「二、项目编写规范 → R-002」

### [P-002-1] 模板内联数组过滤表达式，数据量大时重复计算

- 文件：`src/views/materialFlow/firstMileLogistics/components/confirmWarehous.vue`
- 发现时间：2026/05/12
- 问题描述：`el-table` 的 `:data` 属性直接绑定 `.filter()` 表达式，Vue 每次渲染都会重新执行过滤逻辑，当列表数据量大时性能损耗明显。
- 影响：每次渲染/响应式更新都重新过滤数组，滚动、分页、筛选等操作卡顿。
- 错误示例：

```html
<el-table
	:data="props.currentRow?.deliveryVOList?.filter((item: any) => item.allowIntoQuantity !== 0) ?? []"
></el-table>
```

- 正确示例：

```ts
const filteredDeliveryList = computed(
	() =>
		props.currentRow?.deliveryVOList?.filter(
			(item: any) => item.allowIntoQuantity !== 0
		) ?? []
);
```

```html
<el-table :data="filteredDeliveryList"></el-table>
```

- 对应规范：见「二、项目编写规范 → R-002」

### [P-003] onMounted 重复请求已在 useMultipleDicts 中注册的字典

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：`useMultipleDicts` 已注册 `product_type` 字典，`onMounted` 中又单独调用 `getDicts('product_type')` 并赋值给 `productTypeOptions`，造成同一接口被请求两次。
- 影响：多一次无效网络请求，且维护两份数据源容易造成数据不一致。
- 错误示例：

```ts
const productTypeOptions = ref([]);
onMounted(async () => {
  const res = await getDicts('product_type'); // 重复请求！
  productTypeOptions.value = res.data.map(...);
});
```

- 正确示例：

```ts
// 直接复用 dictMap 中已有的数据，无需额外请求
const productTypeOptions = computed(() => dictMap.value?.productType ?? []);
```

- 对应规范：见「二、项目编写规范 → R-001、R-003」

### [P-004] 模板中对同一数据多次调用普通函数

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：「关联产品/配件」列中，`getMaterialText(row.materials)` 在同一个列模板内被调用了 3 次（`v-if` 判断、`tooltip content`、展示文本各一次），每次渲染都会重复执行相同逻辑。
- 影响：函数执行次数是实际需要的 3 倍，数据量大时累积开销明显。
- 错误示例：

```html
<el-tooltip v-if="getMaterialText(row.materials).length > 25" <!-- 第1次 -->
	:content="getMaterialText(row.materials)"
	<!-- 第2次 -->
	> {{ truncateText(getMaterialText(row.materials), 10) }}
	<!-- 第3次 -->
</el-tooltip>
```

- 正确示例：

```ts
// 在 script 中预处理，或通过模板变量缓存（Vue 3.4+ 支持 v-memo）
// 简单场景：将函数调用结果缓存到局部变量（可结合 row 数据预处理）
```

```html
<!-- 减少重复调用，逻辑收拢到函数内部 -->
<el-tooltip
	v-if="row.materials.length && row.materials[0] && getMaterialText(row.materials).length > 25"
	:content="getMaterialText(row.materials)"
	placement="top"
>
	<div>{{ truncateText(getMaterialText(row.materials), 10) }}</div>
</el-tooltip>
```

- 对应规范：见「二、项目编写规范 → R-002、R-004」

### [P-005] el-tooltip content 绑定内联计算表达式

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：`el-tooltip` 的 `:content` 属性直接绑定 `row.platforms.map(...).join(', ')` 等表达式，即使 tooltip 未展示，Vue 也会在每次渲染时计算该绑定值。
- 影响：tooltip 的 content 计算在每行每次渲染时都执行，与第 P-002 问题叠加，进一步加重渲染负担。

```ts
const formatPlatformTooltip = (platforms: any[]): string =>
	platforms?.map((p) => p.platformText).join(', ') ?? '';
```

```html
<el-tooltip :content="formatPlatformTooltip(row.platforms)"></el-tooltip>
```

### [P-006] 大量使用行内样式，导致样式难以维护且无法复用

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：模板中大量使用 `style="..."` 或 `:style="{}"` 绑定样式，包括 `display: flex`、`flex-direction`、`width`、`overflow` 等高频重复的样式声明，分散在十余处。
- 影响：
  - 相同样式无法复用，代码冗余严重
  - 行内样式优先级高，覆盖困难，主题切换/响应式改造成本高
  - 模板可读性差，结构与样式混杂
  - 无法被 CSS 压缩工具优化
- 错误示例：

```html
<div
	:style="{ display: 'flex', flexDirection: 'column', justifyContent: 'center', alignItems: 'flex-start', width: '100%' }"
></div>
```

```html
<el-select style="width: 220px"></el-select>
```

- 正确示例：

```html
<!-- 通用布局类抽取到 <style scoped> 或 Tailwind 工具类 -->
<div class="col-flex-start">
	<el-select class="w-55"></el-select>
</div>
```

```scss
// <style scoped>
.col-flex-start {
	display: flex;
	flex-direction: column;
	justify-content: center;
	align-items: flex-start;
	width: 100%;
}
```

- 对应规范：见「二、项目编写规范 → R-007」

### [P-007] 在表格列模板中使用 el-row/el-col 布局组件造成冗余 DOM 节点

- 文件：`src/views/product/product-info/sku/index.vue`
- 发现时间：2026/04/08
- 问题描述：`el-table` 列模板中使用 `<el-row>` + `<el-col :span="24">` 做单列布局，实际上 `span="24"` 等于占满整行，`el-row/el-col` 在这里没有任何布局意义，只是额外渲染了 2 个 DOM 节点。每行数据渲染 2 组这样的结构，100 行数据就多出 400 个无效节点。
- 影响：增加 DOM 树深度，加重浏览器 layout/paint 开销，在大数据量列表中影响滚动流畅度。
- 错误示例：

```html
<el-row class="ellipsis-text">
	<el-col :span="24" class="flex">
		<span>{{ row.skuCode }}</span>
	</el-col>
</el-row>
```

- 正确示例：

```html
<!-- 直接用原生 div 或语义标签，配合 Tailwind 类实现相同效果 -->
<div class="flex ellipsis-text">
	<span>{{ row.skuCode }}</span>
</div>
```

- 对应规范：见「二、项目编写规范 → R-009」

## 二、项目编写规范

### R-001 字典/公共数据请求去重

- 使用 `useMultipleDicts` 时，配置数组中每个 `key` 只允许出现一次。
- 注册前先检查是否已有相同 `key`，避免因复制粘贴引入重复项。
- 如果多处需要同一份数据，统一从 `dictMap` 中取，不要再单独发起请求。

### R-002 禁止在模板中写复杂计算表达式

- `el-table` 列模板、`v-for` 循环体内，禁止直接写 `Array.map()`、`Array.filter()`、`Array.join()` 等链式操作。
- 应将逻辑抽取为 `script` 中的函数或 `computed`，模板只做简单调用。
- 判断依据：表达式超过一个操作符，或涉及数组遍历，就应该抽出去。

### R-003 优先复用已有响应式数据，避免重复请求

- 页面中已通过 hook 或 store 获取的数据，直接用 `computed` 派生，不要再用 `onMounted` 重复请求。
- 新增数据需求时，先检查 `dictMap`、store、父组件传参中是否已有，再决定是否发起新请求。

### R-004 同一函数在同一模板作用域内只调用一次

- 如果同一个函数需要在 `v-if`、`:content`、插值 `{{ }}` 中多次使用，应先将结果存入局部变量或利用 `v-memo` 缓存。
- Vue 3.4+ 可使用 `v-memo` 对列表行进行缓存优化，避免不必要的重渲染。

### R-005 TypeScript 类型约束

- 禁止使用 `any` 类型，尤其是在函数参数和返回值上。
- 列表数据、字典数据应定义对应的 `interface`，放在 `src/types/` 目录下。
- 模板中的 `(p: any)` 写法应替换为具体类型。

### R-006 组件懒加载

- 页面级弹窗、非首屏组件统一使用 `defineAsyncComponent` 懒加载，避免首屏 bundle 过大。
- 路由级组件使用 `() => import(...)` 懒加载。

### R-007 禁止使用行内样式

- 模板中禁止使用 `style="..."` 静态行内样式。
- 禁止使用 `:style="{}"` 动态绑定纯静态的样式值（如固定宽度、固定布局）。
- 优先使用 Tailwind CSS 工具类（如 `w-55`、`flex`、`flex-col`、`items-start`）。
- 多处复用的布局组合抽取为 `<style scoped>` 中的语义化 class（如 `.col-flex-start`）。
- 仅允许在样式值需要由 JS 动态计算时使用 `:style`（如根据变量动态设置宽度/颜色）。

```html
<!-- 禁止 -->
<div style="display: flex; flex-direction: column; width: 100%">
	<el-select style="width: 220px">
		<!-- 允许：值来自 JS 变量 -->
		<div :style="{ width: dynamicWidth + 'px' }">
			<!-- 推荐 -->
			<div class="col-flex-start">
				<el-select class="w-55"></el-select>
			</div></div
	></el-select>
</div>
```

### R-008 Tailwind 宽度工具类与 Element Plus 组件配合

- Element Plus 组件（`el-select`、`el-input` 等）设置固定宽度时，使用 Tailwind 的 `w-*` 类而非 `style="width: Xpx"`。
- 常用映射参考：`220px ≈ w-55`、`150px ≈ w-[150px]`、`130px ≈ w-[130px]`。
- 非标准尺寸使用 Tailwind 任意值语法：`w-[Xpx]`。

### R-009 禁止在表格列模板中使用 el-row/el-col 做单列布局

- `el-table` 列模板内，如果只需要一行内容排列（flex 横排或纵排），直接用 `<div>` + Tailwind 类，禁止使用 `<el-row>` + `<el-col>`。
- `el-row/el-col` 仅用于页面级多列栅格布局，不用于表格单元格内的简单排列。
- 判断依据：如果 `el-col` 的 `span` 是 24（占满），或者只有一个 `el-col`，就应该替换为普通 `div`。

```html
<!-- 禁止 -->
<el-row>
	<el-col :span="24" class="flex">
		<span>{{ row.name }}</span>
	</el-col>
</el-row>

<!-- 推荐 -->
<div class="flex ellipsis-text">
	<span>{{ row.name }}</span>
</div>
```

### [P-008] 使用魔法数字/魔法字符串做状态判断，导致可读性差且难以维护

- 文件：`src/views/materialFlow/orderContainer/index.vue`
- 发现时间：2026/04/24
- 问题描述：模板 `v-if` 和 script 逻辑中直接使用 `1`、`2`、`3`、`5`、`7` 等裸数字判断订柜单状态，阅读代码时无法直接理解数字含义，修改时也容易遗漏或写错。
- 影响：可读性差，维护成本高，枚举值变更时需要全局搜索替换，容易遗漏。
- 错误示例：

```html
<!-- 模板中 -->
<el-dropdown-item
	v-if="(row.status === 1 || row.status === 2 || row.status === 3) && auth('orderContainer_edit')"
>
	编辑
</el-dropdown-item>
```

```ts
// script 中
if (row.status !== 2 && row.status !== 3) {
	useMessage().warning('只能操作状态为【已订舱、已发送工厂邮件】的订柜单据');
}
```

- 正确示例：

```ts
// enums.ts — 与页面同目录，或放在 src/enums/ 统一管理
export enum OrderContainerStatus {
	WAIT_ORDERING = 1, // 待订舱
	ALREADY_ORDERING = 2, // 已订舱
	ALREADY_SEND_MAIL = 3, // 已发送工厂邮件
	ALREADY_OUT_STORAGE = 7, // 已出库
	CANCEL = 5, // 已作废
}
```

```html
<!-- 模板中 -->
<el-dropdown-item
	v-if="
    (row.status === OrderContainerStatus.WAIT_ORDERING ||
      row.status === OrderContainerStatus.ALREADY_ORDERING ||
      row.status === OrderContainerStatus.ALREADY_SEND_MAIL) &&
    auth('orderContainer_edit')
  "
>
	编辑
</el-dropdown-item>
```

```ts
// script 中
if (
	row.status !== OrderContainerStatus.ALREADY_ORDERING &&
	row.status !== OrderContainerStatus.ALREADY_SEND_MAIL
) {
	useMessage().warning('只能操作状态为【已订舱、已发送工厂邮件】的订柜单据');
}
```

### R-010 禁止使用魔法数字/魔法字符串做状态判断

- 业务状态值（如订单状态、审核状态、类型标识等）禁止直接在模板或 script 中写裸数字/字符串。
- 应在页面同目录下新建 `enums.ts`，或统一放在 `src/enums/` 目录，用 TypeScript `enum` 定义所有状态值。
- 枚举命名使用大写下划线风格，注释说明每个值的业务含义。
- 模板和 script 中统一引用枚举，不允许出现无注释的裸数字比较。

```ts
// 禁止
if (row.status === 1) { ... }
if (row.status === 2 || row.status === 3) { ... }

// 推荐
import { OrderContainerStatus } from './enums';

if (row.status === OrderContainerStatus.WAIT_ORDERING) { ... }
if (
  row.status === OrderContainerStatus.ALREADY_ORDERING ||
  row.status === OrderContainerStatus.ALREADY_SEND_MAIL
) { ... }
```
