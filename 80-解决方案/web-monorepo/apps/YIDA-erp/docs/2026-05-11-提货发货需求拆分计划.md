# 提货/发货流程优化需求拆分计划（V2）

## 1. 文档目标与边界

- 目标：将本次 UAT 优化需求拆分为可执行的前端改造任务清单，便于按阶段推进与联调。
- 范围：仅覆盖 `qianyi-ui` 前端实现；后端能力（新接口、状态机、飞书通知、定时任务）作为依赖项列出，不在本文展开实现设计；“货件维护国家配置表”为后台表，前端不做配置页面。
- 优先级：先实施 4.1 + 4.2，再实施 4.3 + 4.4。
- 隐藏内容处理：飞书文档未公开部分先标记 `TODO`，待你补充后再细化。

## 3. 需求拆分（按优先级）

### 3.1 P1-A：4.1 货件维护调整（提货计划 + 发货计划 + 订柜管理联动）

#### A1. 提货计划状态流与校验改造（入平台仓）

- 前端改造点
  - 调整提货计划行操作与批量操作的可用条件（按“配置国家/未配置国家”分支）。
  - “批量推送发货（一件代发）”改名为“批量推送发货”，并放开到：
    - 一件代发；
    - 入平台仓且国家未配置货件维护。
  - “批量推送发货（入平台仓）”改名为“批量推送发货（有货件）”，限制仅配置国家可用。
  - 推送发货前置校验按国家配置分支：
    - 配置国家：必须维护货件信息 + 标签；
    - 未配置国家：仅校验运输方式 + 预计发货时间 + 备注。
- 涉及文件（预计）
  - `src/views/mcp/collectionPlan/index.vue`
  - `src/views/mcp/collectionPlan/components/pushDeliverGoods-dialog.vue`
  - `src/views/mcp/collectionPlan/components/preservePackageInfo-dialog.vue`
  - `src/views/mcp/collectionPlan/components/uploadLabel.vue`
- 后端依赖
  - 需提供“货件维护国家配置”查询接口（前端用于判断校验分支）。
  - 提货计划状态机需后端按规则流转，前端仅消费状态并控制按钮显隐。

#### A2. 发货计划新增“维护货件信息/上传标签/批量维护/批量上传标签/提醒运营创建货件”

- 前端改造点
  - 操作列新增“维护货件信息”按钮（权限可配置）。
  - 原“上传标签”按钮可用条件调整为：状态=待拼柜/已拼柜/已订舱，发货方式=入平台仓，且国家未配置。
  - 列表顶部新增批量按钮：
    - 批量维护货件；
    - 上传货件标签（附件命名与格式校验）；
    - 提醒运营创建货件。
  - 批量提醒执行后提示成功并刷新，日志按钮可查看记录。
- 涉及文件（预计）
  - `src/views/materialFlow/deliveryPlan/index.vue`
  - `src/views/materialFlow/deliveryPlan/config/index.ts`
  - `src/views/materialFlow/deliveryPlan/components/upFile.vue`
  - 新增/复用批量维护弹窗组件（同提货计划能力）
  - `src/api/materialFlow/deliveryPlan/index.ts`（补充接口声明）
- 后端依赖
  - 发货计划批量维护货件导出模板接口。
  - 发货计划批量上传标签接口（服务端二次校验命名与必传项）。
  - 提醒运营创建货件接口（后端按运营分组发送飞书卡片）。

#### A3. 订柜管理“发送工厂装柜邮件”前置校验改造

- 前端改造点
  - 点击“发送工厂装柜邮件”前，调用校验接口或复用发送接口错误码。
  - 对校验失败弹出明确提示：`发货计划[xxx]的货件信息或标签未维护，不可操作！`
- 涉及文件（预计）
  - `src/views/materialFlow/orderContainer/index.vue`
  - `src/views/materialFlow/orderContainer/components/postEmailDialog.vue`
  - `src/api/materialFlow/orderContainer/index.ts`
- 后端依赖
  - 校验规则：平台入库单号、箱唛附件、条码附件不能为空。
  - 返回失败明细（发货单号列表）供前端拼接提示。

## 5. 联调与验收清单

### 5.1 关键场景回归

- 提货计划：
  - 配置国家 vs 未配置国家，两套状态与校验路径必须正确。
  - 批量推送发货（无货件/有货件）按钮权限与可选单据范围正确。
- 发货计划：
  - 维护货件、上传标签、批量维护、批量上传、提醒运营仅在限定条件可操作。
  - 标签附件命名与格式校验失败时提示明确。
- 订柜管理：
  - 发送工厂装柜邮件前置校验阻断有效，提示包含具体发货单号。
  - 导出装车单状态限制生效。
- 已发货计划：
  - 默认时间范围与状态展示正确；
  - 导出与列管理可用；
  - 日志可追溯。

### 5.2 数据与权限验收

- 权限点新增后，未授权账号不展示按钮。
- 新字段“运输方式”在新增/批量新增/列表/导出/弹窗保持一致。
- 所有新增按钮与批量能力均有操作日志记录。
