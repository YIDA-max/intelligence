# M1 逐文件开发任务单（4.1 + 4.2）

## 1. 目标与范围

- 目标：完成提货计划与发货计划流程改造，并串联订柜管理发邮件前置校验。
- 范围：仅前端改造；后端接口与状态流转由服务端提供。
- 约束：货件维护国家配置表由后台维护，前端只消费配置结果。

## 2. 交付顺序

1. 提货计划（`collectionPlan`）先落地状态/按钮/校验与运输方式。
2. 发货计划（`deliveryPlan`）落地新增按钮与批量能力入口。
3. 订柜管理（`orderContainer`）接入发送邮件前置校验与导出装车单入口。
4. 联调并完成回归用例与文案核对。

### 3.2 发货计划模块

#### `src/views/materialFlow/deliveryPlan/index.vue`

- 在顶部工具栏新增独立按钮：
  - `批量维护货件`
  - `上传货件标签`
  - `提醒运营创建货件`
- 操作列按钮规则调整：
  - `维护货件信息`：状态=待拼柜/已拼柜/已订舱，发货方式=入平台仓，国家未配置。
  - `上传标签`：同上规则。
- 批量操作校验：
  - 状态、发货方式、国家配置命中必须全量满足；
  - 不满足时提示具体发货单号列表。
- “提醒运营创建货件”执行成功后刷新列表并提示成功。

#### `src/views/materialFlow/deliveryPlan/config/index.ts`

- 如需新增列/按钮配置，在配置文件同步维护。
- 确认状态映射中保留待拼柜/已拼柜/已订舱可操作描述。

#### `src/views/materialFlow/deliveryPlan/components/upFile.vue`

- 增强上传校验：
  - 附件命名：`提货单号-箱唛附件`、`提货单号-条码附件`、`提货单号-透明标签`；
  - 箱唛/条码必传，透明标签非必传；
  - 格式仅允许 `PDF/PNG/JPG`。

- 为新增接口补充参数与返回类型定义。

## 5. 验收用例（M1 最小集合）

- 用例 1：配置国家入平台仓提货计划，必须维护货件与标签后才可推送。
- 用例 2：未配置国家入平台仓提货计划，可在维护运输方式/预计发货时间/备注后推送。
- 用例 3：发货计划在待拼柜/已拼柜/已订舱时，满足条件展示“维护货件信息/上传标签”。
- 用例 4：批量上传货件标签附件命名错误时，拦截并给出具体错误。
- 用例 5：提醒运营创建货件成功后，列表刷新且日志可查。
- 用例 6：订柜管理发送工厂邮件前置校验失败时，提示具体发货单号。
- 用例 7：导出装车单仅在已安排装柜及后续状态可触发，导出走默认模式。
