# 接口对接标准流程文档

## 概述

本文档定义了从 Apifox 导出的接口文档到前端页面配置的标准对接流程，适用于所有类似的接口对接场景。

---

## 输入材料

### 必需材料

1. **Apifox 导出的 JSON 文件** - 包含完整的接口定义
2. **需求文档** - 说明业务需求和页面功能
3. **现有代码结构** - 了解项目的目录结构和命名规范

### 可选材料

- 原型图或 UI 设计稿
- 业务流程说明
- 字段映射关系表

---

## 流程步骤

### 第一步：解析 Apifox 文档

#### 1.1 提取接口结构

```python
# 使用 Python 脚本解析 Apifox JSON
import json

with open('接口文档.apifox.json', encoding='utf-8') as f:
    data = json.load(f)

# 提取模块结构
modules = data['apiCollection'][0]['items'][0]['items']

# 输出接口列表
for module in modules:
    print(f"模块: {module['name']}")
    for api in module['items']:
        method = api['api']['method']
        path = api['api']['path']
        name = api['name']
        print(f"  - {name}: {method.upper()} {path}")
```

#### 1.2 识别关键信息

从 Apifox 文档中提取：

- **模块名称**：`apiCollection.items[0].name`
- **接口名称**：`apiCollection.items[0].items[0].name`
- **接口路径**：`apiCollection.items[0].items[0].api.path`
- **请求方法**：`apiCollection.items[0].items[0].api.method`
- **查询参数**：`apiCollection.items[0].items[0].api.parameters.query`
- **返回数据**：`apiCollection.items[0].items[0].api.responses`

#### 1.3 提取数据模型

从 `schemaCollection` 中提取数据模型定义：

- 查询参数类型（DTO）
- 返回数据类型（VO）
- 分页响应类型

### 第二步：创建 API 接口文件

#### 2.1 确定文件路径

根据业务模块确定 API 文件位置：

```
/src/api/{一级模块}/{二级模块}/{三级模块}/index.ts
```

示例：

- 仓间调拨发货：`/src/api/integrations/shippingIntegration/warehouseTransfer/index.ts`
- 仓间调拨收货：`/src/api/integrations/receivingIntegration/warehouseTransfer/index.ts`

#### 2.2 定义 TypeScript 类型

**查询参数类型：**

```typescript
/**
 * {模块名称} - 分页查询参数
 */
export interface {ModuleName}PageQuery {
	/** 字段说明 */
	fieldName?: string;
	// ... 其他字段

	/** 分页参数 */
	total?: number;
	size?: number;
	current?: number;
	'orders[0].column'?: string;
	'orders[0].asc'?: boolean;
	optimizeCountSql?: boolean;
	searchCount?: boolean;
	optimizeJoinOfCountSql?: boolean;
	maxLimit?: number;
	countId?: string;
	[key: string]: any;
}
```

**返回数据类型：**

```typescript
/**
 * {模块名称} - 列表数据项
 */
export interface {ModuleName}VO {
	/** 字段说明 */
	fieldName?: string;
	// ... 其他字段
}

/**
 * {模块名称} - 分页响应
 */
export interface {ModuleName}PageResponse {
	code?: number;
	msg?: string;
	data?: {
		records?: {ModuleName}VO[];
		total?: number;
		size?: number;
		current?: number;
		orders?: Array<{ column?: string; asc?: boolean }>;
		optimizeCountSql?: boolean;
		searchCount?: boolean;
		optimizeJoinOfCountSql?: boolean;
		maxLimit?: number;
		countId?: string;
	};
}
```

#### 2.3 定义接口函数

**命名规范：**

- GET 请求：`get{功能名称}`
- POST 请求：`post{功能名称}`
- PUT 请求：`put{功能名称}`
- DELETE 请求：`delete{功能名称}`

**示例：**

```typescript
/**
 * {模块名称} - 分页查询
 */
export function get{ModuleName}Page(params?: {ModuleName}PageQuery) {
	return httpService.get<{ModuleName}PageResponse>('/api/path', {
		params,
	});
}

/**
 * {模块名称} - 导出
 */
export function postExport{ModuleName}(data?: {ModuleName}PageQuery) {
	return httpService.post('/api/path/export', data, {
		responseType: 'blob',
	});
}

/**
 * {模块名称} - 生成金蝶单据
 */
export function getGenerate{ModuleName}KingdeeDoc(params?: GenerateKingdeeDocParams) {
	return httpService.get<GenerateKingdeeDocResponse>('/api/path/generate', {
		params,
	});
}

/**
 * {模块名称} - 更新集成数据
 */
export function getUpdate{ModuleName}IntegratedData(params?: GenerateKingdeeDocParams) {
	return httpService.get<GenerateKingdeeDocResponse>('/api/path/update', {
		params,
	});
}
```

### 第四步：对接页面配置

#### 4.1 确定页面路径

根据业务模块确定页面配置文件位置：


```typescript

	get{ModuleName}Page,
	postExport{ModuleName},
	getGenerate{ModuleName}KingdeeDoc,
	getUpdate{ModuleName}IntegratedData,

export const listApi = {
	getList: get{ModuleName}Page,
	exportList: postExport{ModuleName},
	generateList: getGenerate{ModuleName}KingdeeDoc,
	updateList: getUpdate{ModuleName}IntegratedData,

```

#### 4.3 对接查询条件（searchConfig）

**对接规则：**

- 对于接口不支持的字段，保留配置并添加注释
- 不要更改文档顺序以及随意删除、增加字段，对于不清晰的最后输出异常总结

1. **customCascaderApi 组件**

   - 使用 `text` 匹配接口字段
   - 将接口字段名填入 `code`

   ```typescript
   {
     component: 'customCascaderApi',
     props: {
       componentType: 'warehouseId',
       settings: {
         code: 'apiFieldName', // 接口字段名
         text: '页面显示名称',
       },
     },
   }
   ```

2. **TreeSelectInputApi 组件**

   - 使用 `text` 匹配接口字段
   - 将接口字段名填入 `code`

   ```typescript
   {
     component: 'TreeSelectInputApi',
     props: {
       componentType: 'kingdeeOrg',
       settings: {
         text: '页面显示名称',
         code: 'apiFieldName', // 接口字段名
       },
     },
   }
   ```

3. **SelectInput 组件**

   - 使用 `label` 匹配接口字段
   - 将接口字段名填入 `value`

   ```typescript
   {
     component: 'SelectInput',
     props: {
       searchTypes: [
         {
           label: '页面显示名称',
           value: 'apiFieldName', // 接口字段名
         },
       ],
     },
   }
   ```

4. **SelectDate 组件**
   - 使用 `label` 匹配接口字段
   - 将开始时间字段填入 `dateStart`
   - 将结束时间字段填入 `dateEnd`
   ```typescript
   {
     component: 'SelectDate',
     props: {
       searchTypes: [
         {
           label: '页面显示名称',
           value: 'dateFieldName',
           isAddHMS: true,
           dateStart: 'dateFieldNameStart', // 开始时间字段
           dateEnd: 'dateFieldNameEnd', // 结束时间字段
         },
       ],
     },
   }
   ```

#### 4.4 对接显示列（columns）

**对接规则：**

- 使用 `title` 匹配接口返回字段
- 将接口字段名填入 `field`
- 对于接口不支持的字段，保留配置并添加注释
- 不要更改文档顺序以及随意删除、增加字段，对于不清晰的最后输出异常总结

````markdown
## 标准化命名规范

### API 文件命名

- 文件名：`index.ts`
- 路径：`/src/api/{一级模块}/{二级模块}/{三级模块}/index.ts`
- 注意事项：有些模块没有三级模块

### 类型命名

- 查询参数：`{ModuleName}PageQuery`
- 返回数据项：`{ModuleName}VO`
- 分页响应：`{ModuleName}PageResponse`
- 导出数据项：`{ModuleName}ExcelVO`
- 通用参数：`GenerateKingdeeDocParams`
- 通用响应：`GenerateKingdeeDocResponse`

### 函数命名

- 总的命名逻辑 `{请求类型}{api.path的通过‘/’切割最后两个使用驼峰拼接}`
- 分页查询：`get{ModuleName}Page`
- 导出：`postExport{ModuleName}`
- 新增：`post{ModuleName}`
- 修改：`put{ModuleName}`
- 删除：`delete{ModuleName}`
- 详情：`get{ModuleName}Detail`
- 生成单据：`getGenerate{ModuleName}KingdeeDoc`
- 更新数据：`getUpdate{ModuleName}IntegratedData`

### 页面模块配置命名

- 模块标识：`{moduleName}_{pageType}`
  - 示例：`warehouseTransfer_notGeneratedOrder`
  - 示例：`warehouseTransfer_generatedOrder`
  - 示例：`warehouseTransfer_generatedFailedOrder`

## 常见接口模式

### 模式一：标准 CRUD

```typescript
// 分页查询
export function get{ModuleName}Page(params?: {ModuleName}PageQuery) {
	return httpService.get('/api/path', { params });

// 新增
export function post{ModuleName}(data: {ModuleName}DTO) {
	return httpService.post('/api/path', data);
}

// 修改
export function put{ModuleName}(data: {ModuleName}DTO) {
	return httpService.put('/api/path', data);
}

// 删除
export function delete{ModuleName}(id: number) {
	return httpService.delete(`/api/path/${id}`);
}

// 详情
export function get{ModuleName}Detail(id: number) {
	return httpService.get(`/api/path/${id}`);
}
```
````

### 模式二：导出功能

```typescript

export function postExport{ModuleName}(data?: {ModuleName}PageQuery) {
```

## 特殊场景处理

### 场景一：字段名称不匹配

当页面字段名与接口字段名不一致时：

1. 优先使用接口字段名
2. 在注释中说明原页面字段名
3. 记录在字段对接清单中

```typescript
{
	title: '实际收货日期', // 页面显示
	field: 'actualReceiptDate', // 接口字段（原字段：actualShippingDate）
	width: 150,
}
```

### 场景二：接口字段缺失

当接口不提供某个字段时：

1. 保留字段配置
2. 添加注释说明接口不支持
3. 记录在缺失字段清单中

```typescript
{
	title: '产品名称',
	field: 'productName', // 接口不支持，保留字段
	width: 150,
}
```

### 场景三：字段类型转换

当接口返回的字段类型与页面需要的不一致时：

1. 在类型定义中使用正确的类型
2. 在注释中说明需要转换
3. 如需转换，在页面组件中处理

```typescript
export interface ModuleVO {
	/** 状态：0-未生成，1-已生成，2-生成失败（页面需要转换为文本） */
	status?: number;
	/** 状态名称（接口提供，无需转换） */
	statusName?: string;
}
```

### 场景四：多模块共用接口

当多个模块使用相同的接口时：

1. 将接口定义在公共位置
2. 各模块导入使用
3. 如有差异，使用类型扩展

```typescript
// 公共接口
export interface BasePageQuery {
	total?: number;
	size?: number;
	current?: number;
	[key: string]: any;
}

// 模块特定接口
export interface ModulePageQuery extends BasePageQuery {
	moduleSpecificField?: string;
}
```

## 常见问题

### Q1: 接口字段与页面字段不匹配怎么办？

A: 优先使用接口字段名，在注释中说明原字段名，并记录在字段对接清单中。

### Q2: 接口缺少必需字段怎么办？

A: 保留字段配置，添加注释说明接口不支持，记录在缺失字段清单中，并按优先级提交给后端补充。

### Q3: 多个模块使用相同接口怎么办？

A: 将接口定义在公共位置，各模块导入使用。如有差异，使用类型扩展。

### Q4: 接口返回的字段类型不符合预期怎么办？

A: 在类型定义中使用正确的类型，在注释中说明需要转换，在页面组件中处理转换逻辑。

### Q5: 如何处理分页参数？

A: 使用标准的分页参数结构（current, size, total），并包含 MyBatis Plus 的分页参数。
